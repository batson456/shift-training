Tables

If you go back, not even that long ago, tables were the defacto web developer tool. There were one of the only tools that designers had to create structure on a webpage. However, just like flashing colorful gifs and scrolling marquees, they have outlived their usefulness. Now that is not to say that tables are not an important tool, because they still are, but just like all tools, it is important to know when and how to use them.

Don't laugh, you would have done it too!


When not to use

Tables are no longer used for page layouts. They are far too rigid, and do not provide much in the way of flexibility when it comes to CSS styling. If you feel the urge to start wrapping divs in <td> tags, just keep it bottled up. Trust me, it will be much easier to maintain later if your design changes, or if you need to add more content. Plus, would you rather type this?

<table>
  <tbody>
    <tr>
      <td>Left Panel</td>
      <td>Right Panel</td>
    </tr>
  </tbody>
</table>
or this? 
<div>Left Panel</div>
<div>Right Panel</div>

When to use

Tables should be used for what they were designed for. Displaying data in a tabular format. If you have a set of data that you feel would be suited for a table, by all means, insert it in there. This is pretty much the only place where tables should be used in modern web design, especially when it comes to responsive design.


How to use

Tables are very simple to setup, but should require at the very least a few basic elements. I will only go into best practice here, as the specifics of building tables can be found here. If you clicked that link, I am sorry, was just trying to have a little fun. Seriously though, w3cschools has some awesome information on tables if you want to dive into specifics. I just want to go into some of the best practices you can use when building tables.

Here is a basic table structure which I use for almost all of my tables. I always break out the structure into their components, so almost always use a header <thead> and a body <tbody>. Within the header you will use the <tr> tags, just like in the body, but within each row, you should use <th> tags, wherein the body you should use <td> tags. Following these conventions will make life a little easier when trying to style the table.

<table>
  <thead>
    <tr>
      <th>Heading 1</th>
      <th>Heading 2</th>
      <th>Heading 3</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Column 1</td>
      <td>Column 2</td>
      <td>Column 3</td>
    </tr>
  </tbody>
</table>

Just like everything else, there is no golden rule about how everything should be laid out all the time. Feel free to experiment and make your own best practices, HTML is quite forgiving, so placing a tag in an incorrect place usually won't do too much harm, and following a proper structure, although important, is not a necessity, and your pages will usually function just fine, even with a few mistakes.
