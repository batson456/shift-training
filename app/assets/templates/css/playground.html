Clearfix

Items within a div when given floats are removed from the element's box model, which means that although they are within the box, the box does not really recognize them and will not expand to fit. You must apply some sort of clearfix in order to make the box stretch to contain all floated elements. More details can be found here.

 */

    *zoom: 1;
}

Box Sizing

When using divs with a declared width, applying margins or padding to any of the child elements can throw off your layout. In order to fix this, you must do the math to consider the the box model with all margins, borders, paddings, and child elements widths included, or use the box-sizing property. The CSS below applies box-sizing:border-box to all elements in the DOM. More info here.

padding:20px;

CSS

*, *:before, *:after {
	-webkit-box-sizing: border-box;
	-moz-box-sizing: border-box;
	box-sizing: border-box;
}
Note : the box-sizing:border-box property is only available in IE8+

Display Types

There are many different display types you can choose from when creating elements. With these display types you can make any element appear like an element of a completely different type. More info and options here.

CSS

display:block; behaves like a building block, full width and contains boxlike properties
display:inline; behaves like text, it is displayed inline and loses its boxlike properties like a span tag
display:inline-block; is a hybrid between both block and inline display types. It is displaying inline with text, but maintains boxike properties
display:none; is exactly what you would expect, it removes the elemet from the view, although it still exists and the display type can be modified to make it show again

Positioning

Container

CSS

position:relative positioned relative to its spot in the DOM. Remains inline, but can be adjusted using top,bottom,right,left CSS properties based on its initial position in the DOM.
position:absolute is removed from the current page flow and is positioned absoltely to its nearest relatively positioned parent or the document, whichever comes first.
position:fixed behaves like position:absolute in that is removed from the page flow, but is fixed to the viewport and will always remain in the same position.
position:static is the default. It appears to operate like position:relative but can not be moved using top,bottom,left,right CSS properties, and won't be figured when determining absolute positioning of its children.
