Display Types

The display property of an element is probably one of the most important. It allows you to take an existing element, and modify the way it appears on the page. As you saw before, the <div> element generally has a full width and breaks the page layout. If you wanted it to behave more like a <p> tag you could assign the inline display style to it. There are a few different common display types you can use when styling your pages. In order to declare a display type for an element, you must target that element using a CSS selector and assign its display type.

[selector]{
  display: [type]
}

More on display types can be found here. It is important to learn these, because they are probably one of the most used properties you will apply to elements to provide the structure for your page. You can completely change the way an element behaves by modifying its display property.

Floats

One of the other more important properties for providing structure to your page are floats. Although simple in design, implementation of floats can be quite tricky. Basically, floats remove an element from the flow of the page, and "float" them to the right or to the left. Floats are used often when designing web pages. Menus, picture galleries, even page structures are usually designed around floats. This is the primary concept behind tableless web design. Margins and Padding

Margins and padding are very common CSS properties. Margins add spacing around the outside of an element, while padding provides spacing to the inside of an element. The following is a mockup of the box model of an HTML element taken from w3scools.com.



Gotcha : When using padding with an element with a declared with, the margins and padding are added to the width of the element when it gets rendered. Lets say you had a case where you needed an element that is exactly 1000 pixels wide, for example it needs to fit inside a 1000px container. If you declared a 1000 pixel width to the element, and added a margin, padding, or border, the element would expand beyond the bounds of its container. Take the following for example. Simply leaving out the width declaration on this element would fix this problem, as the div element will already expand to fit its container, but if you ever do need to declare a static width to an element, you must account for margins/paddings/borders. There is a property on modern browsers (IE8+) called box-sizing which allows you to specify how margins/paddings should be measured, and whether they should be included as an extension of the box model, or included within the box model. In order to use exclude padding when determining width, use the box-sizing : border-box declaration. Here is the same model as above but with box-sizing: border-box

I find this technique so handy that I almost always include this propery in my documents. It makes calculations much easier, and simplifies life immensely once you start working with fluid designs which make use of percentage based widths as opposed to pixel based. Include this at the top of your CSS file and it will apply the border-box declaration to all elements on your page.

*, *:before, *:after {
  -moz-box-sizing: border-box; -webkit-box-sizing: border-box; box-sizing: border-box;
 }

Frameworks

As you can imagine, there is quite a bit of code that can go into just creating a structure for you page. This is where CSS frameworks or grid systems come in handy. Here is a list of 20 of the more popular frameworks out there, although I must say my two favorites are Foundation and Bootstrap. These provide much more than a simple grid structure for page layouts, they also provide you with a boilerplate for styles including typography, form elements, and a bunch of handy widget elements. I am currently using Bootstrap for the design of this site, and I have literally only declared a dozen or so styles, the rest is all handled by bootstrap. Foundation probably offers the best and most advanced grid system, although it is not entirely cross browser compatible, and will fail miserably on older browsers as it makes extensive use of the box-sizing property.
