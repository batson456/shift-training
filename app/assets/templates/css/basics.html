CSS Basics

What?

CSS is short for Cascading Style Sheets. What CSS allows you to do is organize your style declarations into a hierarchal structure, separated from your view. This allows you to reuse styles, and to change groups of styles without having to dig through your document to find all elements you want to change. CSS and HTML go hand in hand, as CSS brings out the true power of HTML. Lets expand upon our <div> knowledge a little with a simple example of CSS styling, and I think you will begin to see the power. Let's start with a simple unstyled div. Remember, the border is just so you can see it, divs do not normally have borders.

I am a div

Kind of boring looking. In order to change this up, we need to add a class or id attribute to the <div>. These will both help us identify the element within our code, and to apply reusable styles and Javascript hooks to the element. We will go into a little more detail on id and class in a few. Let's apply the myDiv class to this element.

<div class="myDiv">
I am a div  
</div>

Adding this class to the <div> won't do anything by itself. It will still look exactly the same, but we now have a way of targeting it within our stylesheet. We can now apply some styles to this element. Let's say we want it to have a thick blue border, have a pink background, red text, and be smaller and centered on the page, with rounded borders and a shadow. Sounds fancy huh?

I am a div

Now look at how we changed that boring old <div> into something truly beautiful. We could very easily apply this style to multiple elements by simple adding the myDiv class to other elements on the page, and they will all inherit that same look.

How?

Technically speaking, CSS should go into a separate file from the page, denoted by the .css extension. In order to include this file in your page, you simply declare it within a <link> tag in your header. You can also include <style> tags within your page, but this is not as preferred as including separate files.

<link rel="stylesheet" type="text/css" href="mystyle.css">

The beauty behind stylesheets is that styles are interpreted from the top of the file to the bottom, or in the order in which they were declared. This allows you to override your styles for certain pages. Let's say you have a site, with a handful of different pages. You want to have one general style across all pages, but some pages may be a little different and you want to override one aspect of a particular style declaration. In order to acheive this, you would create one site.css and a page.css for each page that you need to provide overrides for. You could then declare them as follows on each page.

<link rel="stylesheet" type="text/css" href="site.css">
<link rel="stylesheet" type="text/css" href="page.css">

If you had a myDiv style in the site.css, and a myDiv style in page.css, any elements with the myDiv hook on your page will inherit the styling from page.css as opposed to site.css. Because the site.css file was declared first, any identical style declarations within the page.css file will override those of the site.css. This is where the cascading part of cascading style sheets comes from. You could also use this structure to keep your CSS more modular. You could have a widgets.css, tables.css, buttons.css which you could then reuse in your other projects.

Syntax

Other than learning all of the different types of style declarations you can use, the syntax is actually quite simple.

[selector]{
  style1 : declaration;
  style2 : declaration;
}

For example, the div that we styled above was done in the following way.

.myDiv{
    border:6px solid blue;
    background:pink;
    width:20%;
    margin:0 auto 20px;
    padding:20px;
    text-align:center;
    color:red;
    border-radius:20px;
    box-shadow:0 0 10px #000;
}

Selectors

In order to use CSS styles, you must have a way of targeting elements on your HTML page. This is where CSS selectors come in. You can target one specific element, groups of elements, or all elements of a particular tag type.

Id

The id attribute is used to target one specific element on a page. There should never be more than one element with the same id on a page. The # symbol will then allow us to target that specific element. If you have a <div id="myDiv"> element, you would target it in your stylesheet as #myDiv. Use id in your stylesheet sparingly, as specific styling is rare. A majority of the time, you should use the class attribute instead.

Class

The class attribute is used to target multiple elements that should inherit the same styling. You can have as many elements with the same class on the same page, but be warned that they will all inherit the same styles if any declarations have been made. If you have a <div class="myDiv"> element, you would target it in your stylesheet as .myDiv. By applying the myDiv class to other elements, they will all inherit the same styling.

Element

What if you want to make all text inputs look different? You could add a class to all input tags, but wouldn't it be easier just to target all input tags within your stylesheet? Well, you are in luck, because this is definitely something that is possible! Within your stylesheet, you could add

input{
  style1 : declaration;
  style2 : declaration;
}

This can be as high level as applying styles to an <html> tag or as low level as applying styles to all <p> tags on a page. You can use all of these style selectors together to provide a fine level of style control. You could have a <div id="myDiv" class="myDivs"> and then this within your stylesheet.

div{
  margin: 20px;
  padding: 5px;
  border:2px solid red;
}
#myDiv{
  color:red;
  text-align:center;
}

.myDivs{
  font-weight:bold;
  font-style:italic;
}

which will produce this output

myDiv

All div elements on the page will have a margin of 20 pixels, 5 pixel padding, and a 2 pixel red border. All div elements on the page with a class of myDivs will be bold and italic. Only the element with an id of myDiv will have red centered text.

Advanced Selectors and techniques

When declaring styles, it can save you quite a bit of headache by being as specific as possible in your declaration. Lets take the following structure for example.

<div id="container">
  <div class="inner">
    <table class="table">
      <thead>
        <tr>
          <th>Header 1</th>
          <th>Header 2</th>
          <th>Header 3</th>
        </tr>
      </thead>
    </table>
  </div>
</div>

If you want to apply a 1 pixel black border to the th elements, you could include.

th{
  border:1px solid black;
}

However, if you include another table on your page, all <th> tags will automatically inherit this border, whether you want it to or not. This is where specificity comes in handy, and why it is generally best practice to be as specific in your style declarations as possible. You can chain selectors by including spaces between them as follows. This will apply the style only to elements falling under the exact structure shown above, and to nothing else. Basically moving down the chain from the outermost element to the exact element that we want styled.

  #container .inner .table thead tr th{
  border:1px solid black;
}

By including spaces in between, you are traversing from the parent to the sibling. However, not including spaces in between allows you to specifically narrow down elements. Let's say you wanted to target specifically an element that has an id of container and the border and red class added to it. Here is how you would do it #container.border.red. You can also select specific elements by class name. For example, you may have a success class which applies the color green to the element. But, what if you want a success <div> to look different than a success >p<, say for instance have a green border and some padding. You could use different class names for each, or, you can simplify your life and apply the same class to both, but change them from your stylesheet. Let's take a look.

.success{
  color:green;
}
div.success{
  padding:10px;
  border:1px solid green;
}
Success Div

Success paragraph

Notice how they are both green, as they both inherited from the success class, but only the <div> had the border and the padding. By adding that extra level of specificity, we are able to use the same class name, but apply different styles based on the element. You can get pretty complex with your selector usage, and I recommend it. It will give you so much more control, and is well worth the learning curve.

You want more?

You also have the option of using pseudo selectors. These are a little more advanced and there are quite a few of them, but here is a handful: :hover, :visited, :first-child, :last-child selectors. These are most commonly used on lists and tables. Lets take an example. You want to have a list of elements with borders on the bottom, but you don't want the last one to have a border. You could add a class to it, or even easier, just use the last-child selector.

.listDemo li{
  border-bottom:1px solid black;
}
.listDemo li:last-child{
  border-bottom:none;
}
