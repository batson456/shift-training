HTML Basics

Basics

HTML or "HyperText Mark-up Language" is a markup language used by all browsers as the structural foundation of webpages. HTML documents are made up of hierarchal sets of tags, or elements. All tags are formed the same way, as code wrapped in angle brackets <code>, and in most cases, each tag will have an accompanying closing tag </code>. It is an incredibly basic language, and once you master a few of the principle concepts, writing HTML can be incredibly simple.

Structure

Here is a basic format that you can use to get kickstarted making HTML pages, and it will provide us with a good starter for a better look into tags. Although this may look confusing at first, we will break it down into chunks in just a moment

<!doctype html>
<html>
  <head>
    <title>Some Title</title>
    <meta charset="utf-8" />
  </head>
  <body>
    All other html inside here
  </body>
</html>

The first tag <!doctype html> is not always required, but it helps determine which type of HTML document your browser will be rendering, and there are different doctype delclarations which you can choose from which are beyond the scope of this lesson. You can find more about doctypes here. The real meat of the page happens between the <html></html> tags. All HTML documents should be wrapped in these tags, and they act as the container for your webpage.

The <head> generally contains descriptive information about your webpage. Inside here you will generally include the <title> of your website, which will usually be displayed in the header of browser and as the title of the page in tab views. You can also include an stylesheet declarations in here, which we will go into greater detail later, as well as any javascript files you may have. The <meta> tags are used to provide greater detail to both the browser, and to search engine crawlers. There is a bunch of information you can include within <meta> tags and you can find more details here.

The <body> tag is the other key component to an html page, and should be included in all pages. Most of the magic happens between these tags, and you should include all content relative information within these tags. Most of the other tags you use should be included inside the body wrapper. None of these tags are absolutely necessary to load a page, but you will have a much easier time learning if you follow this structure. It can definitely save you some layout headaches further down the road.


NOTE: It is usually best practice to include Javascript tags near or at the bottom of an html page within the body tags. Since most Javascript will not run until the page is fully loaded anyway, it can help speed up the load time of the view by putting these tags near the bottom of the document.
