HTML Tags : Crash Course

Paragraphs

In content heavy webpages the <p>, or paragraph tag is probably one of the most widely used. It is simply a wrapper for text. If you examine the code of this page, you will notice quite a few <p> tags. HTML documents do not act quite like standard documents. White space is, other than spaces, almost entirely ignored. Hitting the enter key while writing your markup will have no real impact on the page when it is actually rendered.

line1          
line2
Output:

line1 line2

The proper way of doing this would be

<p>line1</p>
<p>line2</p>
Output:

line1

line2

Each <p> tag you use will wrap all text included within in a single paragraph. To start a new paragraph, simply include another set of <p> tags.

Another incredibly useful tool to help you in setting up your layout is the <br> tag. This is basically the equivalent of hitting the enter key on your keyboard, and will just start a new line, where it is included. If you wanted to output the example above, without using two separate paragraph tags, you could simple insert a <br> tag in between them. Remember that white space is ignored, so the two examples below will provide equivalent output

line1
<br>
line2

line1<br>line2
Output:
line1
line2

Headers

Headers are used just as you would expect. They are a quick and easy way to make a set of text stand out from the rest. A heading tag is creating by an <h followed by a number. Headers are very important and are used by search engine crawlers to help determine what content your page contains. The smaller the number, the larger the heading tag, and the more significant it appears to the search engine crawlers. <h1>,<h2>,<h3>,<h4>.....

Output:

H1

H2

H3

H4

Anchors / Links

Anchors or links are the glue that holds websites together. A link is what allows you to send a user to another page within your website, on someone else's website, or even on the same page. Links can also be used by javascript to perform an action of some sort. An anchor tag is a little trickier than most of the tags we have worked with so far. <a href="">I am a link</a>. The key component here is the href="". This is what tells the browser where to go when the user clicks on the link. Within the quotations, you can include either a link to an external website e.g. http://www.google.com, within your own website using a relative url /home.html, or to a specific point on a page #home. Lets go over a couple examples.

<a href="http://www.google.com">click me</a>
Output:
click me 

<a href="/">click me</a>
Output:
click me

In order to link within a page, you need to provide a destination to link to. This is done by putting another anchor somewhere on your page with a name attribute as follows <a name="destination">. Then to link to this particular portion of the page, you would use <a href="#name">click me </a>

Divs

The single most important tag, which is also the most difficult to grasp, is the <div> tag. This tag is used as the primary building block for all modern websites. Older websites used to use tables to provide structure, but the rigidity of tables makes them ill suited for the fluid nature of most Web 2.0 sites. Without further ado, lets dig into the <div>

A <div> by default acts as a full width block. When used without content, it has no height unless specifically declared, and acts as a sort of wrapper for any content contained within it, and in most cases expands in height to fit all the content contained within. Let's see a div in action. 

<div>
  I am a div
</div>
Output:
I am a div

As you can see, the div stretched to the full width of the container area, while adding no special margins or padding to the information contained inside. Note that I added a border to the <div> so that you could see it clearly. By default, a <div> does not have a border. You will start to truly understand the power behind <div>'s when we start to div into the CSS, which is where we can start manipulatiing them to provide the structure for our page. Because <div> tags are full width by default, placing one in a line of text will usually have the side effect of breaking the text into a new line.

For example <div>this div</div> will break the flow of the text.
Output:
For example
this div
will break the flow of the text.

Spans

A <span> tag is one of the other more important building blocks used in most webpages, and acts as a counterpart to the <div>. The primary difference between a <div> and a <span> is that a span does not have any width to it, and acts an inline piece. A span can be included within a line of text without breaking the layout of the text. <span> tags are more frequently used to provide styling to specific elements in a line of text.

For example <span>this span</span> will not break the flow of the text.
Output:
For example this span will not break the flow of the text.

UL/OL

Last but not least, we have the unordered list <ul> and ordered list<ol> tags. These are probably two of the other most used tags on a webpage. They are not limited to providing data in a list style format, as their name might otherwise signify. Use these tags whenever you want to display similar information in an ordered fashion. For instance, navigation menus, picture galleries, or even just content segments. They provide great structure for handling this information, and are very easy to style. Take this photo gallery for instance, created solely with unordered lists and CSS.

The markup for this is quite simple, which is the beauty of it, although there is quite a bit going on in the CSS that may look a little scary.

<ul class="thumbnails">
    <li>
        <a href="#" class="thumbnail">
          <img src="http://lorempixel.com/300/200" />
        </a>
    </li>
    <li>
        <a href="#" class="thumbnail">
          <img src="http://lorempixel.com/300/200/abstract" />
        </a>
    </li>
    <li>
        <a href="#" class="thumbnail">
          <img src="http://lorempixel.com/300/200/sports" />
        </a>
    </li>
</ul>
